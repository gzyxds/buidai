当您使用 MDC 语法在组件内编写内容和段落时，可以使用 Vue 的 `<slot>` 组件来渲染内容。

## 使用方法

如果您不想修改渲染的内容，只需使用 Vue 的 `<slot>` 组件。

```vue [components/content/Callout.vue]
<template>
  <div class="callout">
    <slot />
  </div>
</template>
```

现在让我们在 Markdown 中使用它：

```mdc [content/index.md]
::callout
这是一个提示框。
::
```

渲染的 HTML 将是：

```html
<div class="callout">
  <p>这是一个提示框。</p>
</div>
```

这种用法类似于使用原生的 `<slot>` 组件。

### 解包

`mdc-unwrap` 属性允许您从渲染的内容中移除一个或多个包装元素。当您想要提取嵌套在原生 Markdown 语法中的内容时，这很有用。每个指定的标签将从 AST 中移除。

让我们从上一个示例中解包 `<p>` 元素：

```vue [components/content/Callout.vue]
<template>
  <div class="callout">
    <slot mdc-unwrap="p" />
  </div>
</template>
```

现在渲染的 HTML 将是：

```html
<div class="callout">
  这是一个提示框。
</div>
```

### 命名插槽

`name` 属性允许您按名称绑定插槽。当您想要渲染不是默认插槽的插槽时，这很有用。

让我们改进 `Callout` 组件，使其具有 `title` 插槽：

```vue [components/content/Callout.vue]
<template>
  <div class="callout">
    <h2 v-if="$slots.title">
      <slot name="title" mdc-unwrap="p" />
    </h2>
    <slot />
  </div>
</template>
```

现在让我们在 Markdown 中使用它：

```mdc [content/index.md]
::callout
#title
请小心！
#default
使用 MDC 和 Vue 组件会让人上瘾。
::
```

这将产生：

```html
<div class="callout">
  <h2>请小心！</h2>
  <p>使用 MDC 和 Vue 组件会让人上瘾。</p>
</div>
```

当不使用 `title` 插槽时，`h2` 元素将不会被渲染。

## 属性

- `mdc-unwrap`: 是否解包内容。当您想要提取嵌套在原生 Markdown 语法中的内容时，这很有用。每个指定的标签将从 AST 中移除。
  - **类型:** `boolean` 或 `string`
  - **默认值:** `false`
  - **示例:** `'p'` 或 `'ul li'`
