title: Cloudflare Workers
description: 将您的 Content 应用部署到 Cloudflare Workers
---

::card
快速设置

1. 使用 `cloudflare_module` 预设和 `2024-09-19` 或更高版本的兼容性日期。
2. 创建一个 D1 数据库并在 Cloudflare Dashboard 中将其连接到您的项目，使用 `DB` 绑定名称，然后在 `nuxt.config.ts` 文件中配置数据库配置。
3. 构建并部署您的应用
::

:hr

Nuxt Content 模块与 [Cloudflare Workers](https://workers.cloudflare.com) 有内置集成，可用于部署您的内容。

该模块将自动检测构建目标并为 Cloudflare Workers 准备必要的配置。

您所需要做的就是创建一个 Cloudflare D1 数据库并将其连接到您的项目。创建 D1 数据库后，您应该
在 `nuxt.config.ts` 文件中使用 `nitro.cloudflare.wrangler.d1_databases` 选项定义数据库配置。

默认情况下，模块将使用 `DB` 绑定名称。您可以通过在 `nuxt.config.ts` 中提供您自己的数据库配置来覆盖数据库配置。

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  compatibilityDate: "2025-05-15",
  nitro: {
    preset: 'cloudflare_module',
    cloudflare: {
      deployConfig: true,
      wrangler: {
        d1_databases: [
          {
            binding: 'DB',
            database_name: 'database-name',
            database_id: '*********-***-****-****-*********'
          }
        ]
      },
    },
  }
})
```

::note
如果您想使用不同的绑定名称，可以通过在 `nuxt.config.ts` 中提供您自己的数据库配置来覆盖数据库配置。查看 [数据库配置](/docs/getting-started/configuration#d1)
::

::note
要将 Nuxt 项目部署到 Cloudflare Workers，您需要使用 `2024-09-19` 或更高版本的兼容性日期。
::

使用 `nuxi build` 命令构建您的项目，然后您可以使用 `wrangler deploy` 命令部署项目。

```bash
npx wrangler deploy
```

就这样！:tada:
