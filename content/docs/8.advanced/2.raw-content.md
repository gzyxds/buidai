---
title: 原始内容
description: 在应用程序中访问内容的原始数据
---

在 Content 版本 2 中，有很多关于在生产环境中访问内容原始数据的请求。在 Content 版本 3 中，可以将内容的原始数据部署到生产环境。

为了将原始内容部署到生产环境，您需要在集合的 schema 中定义 `rawbody` 字段。就这样。

Nuxt Content 将检测您 schema 中的这个神奇字段，并用原始内容填充它。

```ts [content.config.ts]

import { defineCollection, defineContentConfig } from '@nuxt/content'
import { z } from 'zod'

export default defineContentConfig({
  collections: {
    docs: defineCollection({
      source: '**',
      type: 'page',
      schema: z.object({
        rawbody: z.string()
      })
    })
  }
})
```

您可以使用 `queryCollection()` 来获取原始内容。

```vue [pages/index.vue]
<script setup lang="ts">
const route = useRoute()
const { data } = useAsyncData('page-' + route.path, () => queryCollection('docs').path(route.path).first())
</script>

<template>
  <pre>{{ data.rawbody }}</pre>
</template>
```

如果您不想部署特定文件的原始内容，可以在该文件的前置元数据（frontmatter）中添加 `rawbody: ''`。`rawbody` 的自动填充值就像默认值一样，当您在前置元数据中定义 `rawbody` 时，它将被覆盖。

```md [content.md]
---
title: My page
rawbody: ''
---

```

::callout
重要的是要用集合 schema 中定义的相同类型的数据填充前置元数据字段。在这种情况下，`rawbody` 是一个字符串，您应该考虑传递空字符串。不要使用 `boolean` 或其他类型的值。
::
